# 감사조서(워크페이퍼) 자동화

**매 결산기마다 반복 작성하는 감사조서를, PBC(고객 제공) 원본 자료에서 자동으로 조립하고 — 자동화가 못 채우는 항목은 근거자료와 함께 검토자에게 넘겨 "클릭 한 번으로 반영"하게 만드는 도구입니다.**

이 저장소에는 표본 데이터(`sample_data/`)가 포함되어 있어, 클론한 뒤 별도 자료 없이 바로 전체 흐름을 돌려볼 수 있습니다.

핵심 목표는 두 가지입니다.

1. 반복적인 조서 작성 시간을 실제로 줄인다.
2. "1단계 자동 조립은 표준 템플릿을 전제로 하고, **2단계 검토 워크플로우는 계정 종류에 묶이지 않는 재사용 가능한 패턴**"이라는 확장성을 코드로 증명한다.

## 눈여겨볼 점

- **자동 조립 + 사람 검토의 결합**: `openpyxl`로 24개 시트를 실수식으로 조립(`core/`)하고, 자동화가 못 채우는 항목은 React 검토 UI(`frontend/`)로 근거와 함께 넘깁니다.
- **수식 추적 기반 근거 연결**: 조서의 셀을 클릭하면 그 셀 수식이 참조하는 원본 PBC 시트·행을 역추적해 보여줍니다. **행 번호를 하드코딩하지 않아** 다음 기수에 종목/거래 수가 바뀌어도 깨지지 않습니다.
- **"AI 제안 → 사람 승인 → 반영" 흐름**: AI가 근거 문서(계약서)에서 관련 조항을 찾아 "이렇게 반영할까요?"라고 제안하고, 사람이 **예**를 눌러야 그 값이 조서에 반영됩니다. AI가 지어내지 않고 최종 판단은 사람이 합니다.
- **정직한 자동화**: 원천 자료가 없어 기계적으로 못 채우는 항목은 억지로 채우지 않고 "검토 필요"로 남기며, 실제 동작과 데모용 구성을 코드·화면에서 구분합니다.

## 화면 흐름 (React + FastAPI, 3단계)

1. **1단계 — 초벌 조립**: PBC 원본 CSV 11개 + 전기 조서를 드래그로 한 번에 넣고 실행하면 `build_workpaper()`가 실제로 돌아 24개 시트를 조립하고 2단계로 넘어갑니다. (업로드 없이 "샘플 데이터로 실행"도 가능. 1단계를 실행하기 전에는 2·3단계로 넘어갈 수 없습니다.)
2. **2단계 — 검토**: 왼쪽에 조립된 조서 시트, 오른쪽에 근거자료. 파란 배경 셀/블록을 클릭하면 근거가 뜹니다.
   - **원본 PBC 표 대사** — C2 자산·C3 수익·평가액검증(주식/수익증권/선물) → 국내주식명세·채권명세 등 원본 CSV
   - **전기 재무제표** — C1 정산표·재무제표검증 → 재구성한 전기 재무상태표·손익계산서·자본변동표
   - **AI 문서 검색(승인형)** — 특수관계자거래·담보제공자산 → 합성 계약서에서 근거를 찾아 제안하고, 예/아니오로 반영
3. **3단계 — 완료**: 최종 워크북 다운로드 + 검토 대상 항목 요약.

> 화면 캡처는 `docs/`에 이미지를 추가해 이 자리에 넣을 수 있습니다.

## 핵심 설계 결정

출력 워크북은 매번 빈 `openpyxl.Workbook()`에서 새로 조립되며, 각 시트의 출처가 항상 명시적입니다.

- **원본 그대로 복사(`copy_sheet`) vs PBC 재조립**: 외부에서 신뢰하는 시트(`C1_정산표`, `재무제표검증`, `KRS/KRX` 시세표, `통합기준가격대장` 등)는 그대로 복사하고, 나머지(`C2_자산부채평가`·`C3_수익`·`C5_F.N` 주석 등)는 원시 CSV를 파싱해 **실수식으로** 다시 만듭니다. 전기 조서는 스펙일 뿐 정답으로 쓰지 않습니다.
- **라이브 수식**: 지금 잔액이 0인 항목도 실제 값이 생기면 자동 반영되도록 상수 대신 수식을 씁니다.
- **행 개수 독립**: 종목/채권/파생상품처럼 개수가 변하는 목록은 N개 실데이터 행 + 여유 공백 3행 다음에 합계를 둡니다(`core/c5_styles.py::BUFFER_ROWS`). 다음 기수에 항목 수가 늘거나 줄어도 시트를 다시 만들 필요가 없습니다.
- **못 채우는 건 비워두고 검토로**: 특수관계자거래·담보제공자산처럼 원천 자료가 없는 항목은 구조만 만들고 값은 비운 뒤, 사유를 `build_workpaper()`의 `needs_review` 목록에 담아 검토 UI로 넘깁니다. 엑셀 파일 자체에는 "검토 필요" 같은 표시를 넣지 않습니다.

## 실행 방법

### 조립 엔진만 (웹 없이)

```bash
pip install -r requirements.txt
python run_sample_demo.py    # sample_data/ -> sample_output/조립결과.xlsx
```

`sample_data/`의 표본 입력을 읽어 결과 워크북을 생성합니다. 클론 후 별도 자료 없이 바로 돌려볼 수 있습니다.

### 검토 워크플로우 웹앱 (3단계)

```bash
pip install -r requirements.txt
python -m uvicorn api:app --port 8000 --reload   # 백엔드
cd frontend && npm install && npm run dev        # 프론트엔드 (http://localhost:5173)
```

> **AI API 사용량 제한에 대하여**: 승인형 항목(특수관계자거래·담보제공자산)의 근거검색·반영 제안은 실제 Gemini API(임베딩 검색 + 원문 근거 기반 제안 생성)로 구현되어 있습니다. 다만 무료/제한된 API 키에서는 사용량 제한(rate limit)에 걸리면 실시간 AI 호출이 실패할 수 있어, 기본값은 **결정론적 폴백**(키워드 매칭 검색 + 원문에 실제로 있는 내용만 담아 손으로 작성한 제안)입니다. 예/아니오 반영 흐름은 폴백에서도 그대로 동작하고, 화면에 "현재 사용량 제한으로 사전 정의된 제안을 사용 중"이라고 표시됩니다. **AI 기능 자체는 구현되어 있으며**, 할당량이 넉넉한 키가 있으면 `USE_LIVE_AI=1` 환경변수로 실시간 AI를 켤 수 있습니다.

## 코드 구조

```
core/               파싱 · 조립 · 스타일링 로직 (assemble.py가 진입점)
sample_data/         표본 입력 (11개 CSV + 전기 조서)
sample_output/       위 표본으로 만든 결과물 예시
frontend/, api.py    3단계 검토 워크플로우 — React + FastAPI
run_sample_demo.py   sample_data/로 조립 엔진 실행
```

## 알려진 한계

- `특수관계자거래`, `담보제공자산`: 원천 자료 자체가 없어(관계사 목록·대용현황 등) 구조만 생성하고, 값은 검토 대상(승인형)으로 남깁니다.
- KRS/KRX 시세 실시간 연동은 하지 않음 — 전기 조서의 시세를 그대로 복사(외부 데이터 소스 연동은 향후 과제).
- `설정해지내역`(자펀드 분할), `가중평균좌수` 일부는 예시 데이터 부족으로 보류.
- 2단계 시트 값 계산은 Excel COM(pywin32, Windows) 재계산에 의존합니다 — 리눅스 배포 시에는 순수 파이썬 수식계산 또는 사전 계산본 서빙으로 대체가 필요합니다.
