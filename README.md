# 감사조서 자동화

펀드 결산 때마다 비슷한 감사조서(워크페이퍼)를 원본 자료를 보고 손으로 다시 만드는 일은 반복적이고 시간이 오래 걸립니다. 이 도구는 그 초벌 조서를 원본 데이터에서 자동으로 만들어 주고, 자동화가 채우지 못하는 부분은 근거자료와 함께 보여줘서 검토자가 클릭 몇 번으로 처리하게 해 줍니다.

세 단계로 씁니다.

## 1단계 — 초벌 조립

PBC 원본자료(11개 CSV + 전기 조서)를 넣고 실행하면 24개 시트짜리 조서가 자동으로 만들어집니다. 표본 데이터가 들어 있어서, 클론한 뒤 아무 자료 없이 "샘플 데이터로 실행"만 눌러도 전체 흐름을 볼 수 있습니다.

![1단계 업로드](docs/screenshots/1_upload.png)

## 2단계 — 검토

자동으로 만들어진 조서를 왼쪽에, 그 근거자료를 오른쪽에 놓고 검토합니다. 두 가지가 핵심입니다.

**셀을 누르면 그 숫자가 어디서 왔는지 바로 보여줍니다.** 조서의 어떤 셀이든 클릭하면, 그 값이 참조한 원본 PBC 표(또는 전기 재무제표)를 역추적해 오른쪽에 띄웁니다. 감사에서 시간이 많이 드는 "이 숫자 근거가 뭐지" 확인을 그 자리에서 끝냅니다.

![셀 클릭 시 원본 근거 연결](docs/screenshots/3_sheet_evidence.png)

**자동으로 못 채우는 항목은 AI가 근거를 찾아 제안하고, 사람이 승인합니다.** 특수관계자거래·담보제공자산처럼 원본 데이터가 없어 기계적으로 채울 수 없는 항목은, AI가 관련 계약서를 찾아 "이렇게 반영할까요?"라고 제안합니다. 검토자는 원문을 확인하고 예/아니오만 누르면 그 값이 조서에 반영됩니다. AI가 지어내는 게 아니라, 최종 판단은 항상 사람이 합니다.

![AI 제안 후 승인](docs/screenshots/2_review_item.png)

## 3단계 — 완료

검토가 끝난 조서를 내려받고, 아직 손봐야 할 항목을 한눈에 정리해 보여줍니다.

![완료 및 다운로드](docs/screenshots/4_complete.png)

## 실행 방법

저장소에 표본 데이터가 함께 있어서 별도 자료 없이 바로 돌려볼 수 있습니다.

```bash
pip install -r requirements.txt

# (a) 조립 엔진만 돌려보기 — sample_data/ → sample_output/조립결과.xlsx
python run_sample_demo.py

# (b) 검토 워크플로우 웹앱 (3단계)
python -m uvicorn api:app --port 8000 --reload    # 백엔드
cd frontend && npm install && npm run dev         # 프론트엔드 (http://localhost:5173)
```

## 만들면서 신경 쓴 것

- **다음 기수에도 안 깨지게.** 종목·거래 수가 바뀌어도 시트를 다시 짤 필요가 없도록, 행 번호를 하드코딩하지 않고 수식과 구조로 처리했습니다.
- **근거 연결이 계정 종류에 안 묶이게.** 셀의 수식을 역추적하는 방식이라, 항목마다 따로 규칙을 만들지 않고 하나의 로직으로 재사용됩니다. (1단계 자동 조립은 조서 템플릿에 맞춰야 하지만, 2단계 검토 흐름은 이렇게 범용으로 갑니다.)
- **못 채우는 건 억지로 채우지 않기.** 원천 자료가 없는 항목은 값을 지어내지 않고 "검토 필요"로 남겨, 사람이 근거를 보고 판단하게 합니다.

## AI 사용량에 대하여

승인형 항목의 근거검색·제안은 실제 Gemini API로 구현되어 있습니다. 다만 무료/제한된 키에서는 사용량 제한에 걸려 실시간 호출이 실패할 수 있어, 기본값은 사전 정의된 결정론적 제안(원문에 실제로 있는 내용만 담아 작성)입니다. 이때도 예/아니오 반영 흐름은 그대로 동작하고, 화면에도 그렇게 표시됩니다. 넉넉한 키가 있으면 `USE_LIVE_AI=1`로 실시간 AI를 켤 수 있습니다.

## 기술

Python(openpyxl)으로 짠 조립 엔진 + FastAPI 백엔드 + React 프론트엔드.

```
core/               조서 파싱·조립 로직 (assemble.py가 진입점)
api.py, frontend/   3단계 검토 워크플로우
sample_data/        표본 입력
sample_output/      표본으로 만든 결과물 예시
run_sample_demo.py  조립 엔진 실행
```

## 알려진 한계

- 특수관계자거래·담보제공자산: 원천 자료가 없어 구조만 만들고 값은 검토 대상으로 남깁니다.
- 시세는 전기 조서의 값을 그대로 씁니다(외부 데이터 소스 실시간 연동은 향후 과제).
- 2단계 시트 값 계산은 Excel COM(pywin32, Windows) 재계산에 의존합니다 — 리눅스 배포 시에는 대체가 필요합니다.
